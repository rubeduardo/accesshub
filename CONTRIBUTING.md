# Contribuindo para AccessHub

Primeiramente, obrigado por considerar contribuir para o AccessHub! É pessoas como você que tornam AccessHub uma ferramenta tão útil.

## Como Contribuir

### Reportando Bugs

Antes de criar relatórios de bugs, verifique se o problema já foi reportado. Quando criar um relatório de bug, inclua o máximo de detalhes possível:

- **Use um título descritivo**
- **Descreva os passos exatos que reproduzem o problema**
- **Descreva o comportamento observado**
- **Explique o comportamento esperado**
- **Inclua screenshots se possível**

### Sugestões de Melhorias

Sugestões de melhorias são bem-vindas! Para sugerir uma melhoria:

- **Use um título descritivo**
- **Forneça uma descrição detalhada da sugestão**
- **Inclua exemplos**
- **Explique por que isso seria útil**

### Pull Requests

- Siga o guia de estilo do projeto
- Inclua testes apropriados
- Atualize a documentação conforme necessário
- Fim de arquivo em uma nova linha

## Guia de Estilo

### Git Commit Messages

- Use o modo imperativo ("Add feature" não "Added feature")
- Limite a primeira linha a 72 caracteres ou menos
- Referencie issues e pull requests liberalmente depois da primeira linha

### Código

- Siga as convenções do projeto
- Use nomes descritivos para variáveis e funções
- Documente seu código
- Escreva testes
